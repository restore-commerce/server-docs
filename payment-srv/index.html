Payment Service

A microservice for handling payment authorizations and purchases.

Features

  • Supports 100+ payment providers. (Full List)

  • Utilizes Active Merchant as the underlying platform.

  • Exposes a gRPC interface for handling payment specific functionality.

  • Supports both authorization and purchase style payment flows.

Configuration

Purchase flow

The purchase flow is the simpler option.

It goes through the following steps:

  1. Setup the purchase flow (SetupPurchase method)

  2. User redirection to the payment platform page

  3. After user returns from payment platform, complete the payment (Purchase method)

It does not require a checkout or approval page on the store page.

Authorization Flow

The authorization flow utilizes the option that after returning from the payment platform the user can do a final review of the invoice before it gets approved.

It goes through the following steps:

  1. Setup the authorization flow (SetupAuthorization method)

  2. User redirection to the payment platform page

  3. After user returns from payment platform, approve the authorization (Authorize method)

  4. Show the full invoice with an approval button to the user.

  5. After user has approved, complete the payment (Capture method)
